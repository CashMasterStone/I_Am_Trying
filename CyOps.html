2. 1-bit polarity flags and instruction line format

Every “instruction line” in your [CODE] block will look like this:

`text

[Polarity][MNEMONIC].[DOMAIN] : natural-language directive

`

Where:

- Polarity bit (1-bit):

  - + = stabilize, conserve, structure, clarify.

  - - = test, perturb, invert, stress-test, expose limits.

- Mnemonic: The operation (e.g. CONSERVE, ENTROPY, TRACE).

- DOMAIN: The domain focus (PHY, BIO, CHEM, META, NUM, COS, HIST, CIV, CODE).

- Directive: Human-language details for Claude.

Example:

`text

+CONSERVE.PHY : Track what quantities remain invariant in this system over time.

-ENTROPY.COS : Explore how disorder or dispersion increases and what structures resist it.

`

Use one instruction per line. Claude should treat each line as a composable op, executed in order.

3. HEADER format (mode, depth, style switches)

Define your HEADER like this:

`text

STANCE: NEUTRAL | ADVOCATE[X] | CRITIC[X]

- MODE:

  - DETERMINISTIC: Step-by-step, explicit, minimal speculation.

- DEPTH:

  - 2: Structured but compact.

  - 3: Deep, multi-layered explanation.

  - 4: Exhaustive, reference-grade.

- STYLE:

  - TECHNICAL: Equations, mechanisms, formal structure.

  - MYTHIC: Archetypes, symbolism, story-framing.

  - CIVIC: Policy, institutions, material impact.

  - PERSONAL: Directly tied to lived experience.

  - HYBRID: Mix; you can specify combos like TECHNICAL+MYTHIC.

- STANCE:

  - NEUTRAL: Map the terrain.

  - ADVOCATE_X: Argue for principle X (e.g. sovereignty, transparency).

  - CRITIC_X: Stress-test or challenge X.

- CONSTRAINTS:

  - No fluff; every sentence must add new information.

  - Expose assumptions explicitly.
